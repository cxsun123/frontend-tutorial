06 - 响应式设计

视口设置、媒体查询、流式布局、响应式图片/视频与移动端适配。

什么是响应式设计？

响应式 Web 设计（Responsive Web Design, RWD）是一种让网页在所有设备上都能良好呈现的设计方法。 无论用户使用手机、平板还是桌面电脑访问，页面的布局、字体大小、图片尺寸都会自动适应当前屏幕，无需为每种设备单独开发一个网站。
📱

手机（≤ 480px）

单列布局，文字自动缩小，图片自适应宽度

📋

平板（481-1024px）

双列或三列布局，间距适中

🖥️

桌面（≥ 1025px）

多栏布局，内容区最宽，侧边栏展开

↑ 以上三个卡片本身就是一个响应式网格 — 缩小窗口试试，它们会自动堆叠 ↑ 

/* 响应式设计的三大核心技术 */

1. 流式布局 (Fluid Grid)
   使用相对单位 (%, rem, vw, fr) 而非固定像素
   容器宽度随屏幕自动伸缩

2. 弹性媒体 (Flexible Media)
   图片和视频设置 max-width: 100%
   防止内容溢出容器

3. 媒体查询 (Media Queries)
   @media 条件 { CSS规则 }
   在不同屏幕宽度下应用不同的样式

为什么需要响应式设计？

❌ 没有响应式

  • • 固定 960px 宽度 → 手机需要左右滚动
  • • 桌面大图直接加载 → 浪费手机流量
  • • 小字在手机上看不清 → 用户体验差
  • • 可能需要维护两套网站 → 开发成本高
  • • SEO 不友好 → 搜索引擎降低排名

✅ 有响应式

  • • 一套代码适配所有设备 → 维护简单
  • • 布局自动伸缩 → 无需滚动查看
  • • 图片按需加载 → 省流量、加载快
  • • 字体自适应 → 任何屏幕都易读
  • • 搜索引擎统一索引 → SEO 友好
关键数据： 全球超过 60% 的网页流量来自移动设备。 Google 自 2019 年起采用移动优先索引（Mobile-First Indexing）， 即用移动端版本来评估和排名网站。没有响应式设计意味着在搜索引擎中处于劣势。 
/* 响应式 vs 非响应式对比 */

非响应式（旧做法）：
  <!-- 为手机单独建站 -->
  m.example.com  ← 手机用户跳转到 m 站
  www.example.com ← 桌面用户访问原站
  → 需要维护两套代码、两套内容

响应式（现代做法）：
  一套 HTML + CSS + 媒体查询
  同一 URL，不同设备看到不同布局
  → 一套代码、一套内容、自动适配

1. 视口 Meta 标签

所有响应式网页必须在 <head> 中添加：
<meta name="viewport" content="width=device-width, initial-scale=1.0">
它告诉浏览器：视口宽度 = 设备宽度，并以 100% 缩放渲染。没有它，移动端会按 980px 桌面视口渲染。
📱

width=device-width

视口宽度等于设备实际宽度

🔍

initial-scale=1.0

初始缩放比例为 1:1

🚫

user-scalable=no

禁止用户缩放（不推荐）

📐

min/max-scale

限制缩放范围 0.5~2.0

<head>
  <meta name="viewport"
        content="width=device-width, initial-scale=1.0">
</head>

/* 常见 viewport 属性 */
width=device-width    设备宽度
initial-scale=1.0    初始缩放
maximum-scale=2.0    最大缩放
minimum-scale=0.5    最小缩放
user-scalable=yes    是否允许缩放

2. 媒体查询（Media Queries）

媒体查询根据设备特征（宽度、高度、方向等）有条件地应用 CSS 规则。
@media 规则在满足条件时才生效，是响应式设计的核心技术。
📱 移动端（< 600px）
📋 平板端（600px ~ 999px）
🖥️ 桌面端（≥ 1000px）

← 试试调整浏览器窗口宽度，观察上方色块变化 → 

/* 基本语法 */
@media 媒体类型 and (条件) {
  /* CSS 规则 */
}

/* 示例：小于 600px 时背景变蓝 */
@media only screen and (max-width: 600px) {
  body { background-color: lightblue; }
}

/* 示例：平板端特定样式 */
@media only screen and (min-width: 600px) and (max-width: 999px) {
  .container { width: 95%; }
}

/* 示例：桌面端样式 */
@media only screen and (min-width: 1000px) {
  .container { max-width: 1200px; margin: 0 auto; }
}

/* 媒体类型 */
all        所有设备（默认）
screen     屏幕设备
print      打印预览
speech     屏幕阅读器

3. 断点与移动优先（Mobile First）

移动优先：先写移动端样式（无媒体查询），再用 min-width 向上增强。
这比从桌面端开始用 max-width 缩减更符合渐进增强原则。
推荐断点：480px（手机）、768px（平板）、1024px（桌面）、1200px（大屏）。

📱 手机

≤ 480px · 单列布局 · 紧凑间距

📱 大手机

481-768px · 单列或双列 · 适中间距

📋 平板

769-1024px · 双列布局 · 充足间距

🖥️ 桌面

1025-1200px · 三列布局 · 宽松间距

🖥️ 大屏

≥ 1201px · 三列+侧栏 · 最大宽度

⚙️ 工具

Chrome DevTools 可模拟各种设备尺寸

/* ── 移动优先写法（推荐）── */
/* 基础样式：手机 */
.card-grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 12px;
}

/* 平板及以上 */
@media (min-width: 768px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr);
    gap: 16px;
  }
}

/* 桌面及以上 */
@media (min-width: 1024px) {
  .card-grid {
    grid-template-columns: repeat(3, 1fr);
    gap: 20px;
  }
}

/* ── 桌面优先写法（不推荐）── */
.card-grid {
  grid-template-columns: repeat(3, 1fr);  /* 桌面 */
}
@media (max-width: 767px) {
  .card-grid {
    grid-template-columns: 1fr;  /* 手机覆盖 */
  }
}

4. 响应式网格布局

使用 CSS Grid 的 repeat() + auto-fill/auto-fit + minmax() 可以实现无需媒体查询的响应式网格。浏览器自动根据容器宽度决定列数。

卡片 A

自动适应容器宽度

卡片 B

无需媒体查询

卡片 C

自动换行排列

卡片 D

间距由 gap 控制

卡片 E

宽度 >= minmax 最小值

卡片 F

宽度 <= minmax 最大值

/* ── auto-fill：自动填充尽可能多的列 ── */
.auto-fill-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 16px;
}

/* ── auto-fit：拉伸填充可用空间 ── */
.auto-fit-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 16px;
}

/* 区别：auto-fill 保留空列，auto-fit 拉伸填满 */

/* ── 12 列响应式网格（经典）── */
.row { display: flex; flex-wrap: wrap; gap: 16px; }
.col-4  { flex: 0 0 calc(33.333% - 11px); } /* 4/12 = 33% */
.col-6  { flex: 0 0 calc(50% - 8px); }      /* 6/12 = 50% */
.col-12 { flex: 0 0 100%; }                  /* 12/12 = 100% */
@media (max-width: 768px) {
  .col-4, .col-6 { flex: 0 0 100%; }        /* 手机全部单列 */
}

5. 响应式图片

响应式图片的核心：max-width: 100% 让图片不超过容器宽度，height: auto 保持宽高比。
使用 <picture> 元素和 srcset 属性可以根据屏幕尺寸加载不同分辨率的图片。
✅ max-width: 100%（响应式）
❌ 固定宽度 600px（溢出容器）
/* ── 基础响应式图片 ── */
img {
  max-width: 100%;   /* 不超过容器 */
  height: auto;      /* 保持宽高比 */
}

/* ── <picture> 元素：不同屏幕加载不同图片 ── */
<picture>
  <source srcset="img-large.jpg"  media="(min-width: 1024px)">
  <source srcset="img-medium.jpg" media="(min-width: 600px)">
  <img src="img-small.jpg" alt="响应式图片">
</picture>

/* ── srcset 属性：浏览器自动选择最佳图片 ── */
<img src="img-800.jpg"
     srcset="img-400.jpg 400w,
             img-800.jpg 800w,
             img-1200.jpg 1200w"
     sizes="(max-width: 600px) 100vw,
            (max-width: 1024px) 50vw,
            33vw"
     alt="自适应图片">

/* ── 响应式视频 ── */
.video-wrapper {
  position: relative;
  padding-bottom: 56.25%;  /* 16:9 比例 */
  height: 0;
  overflow: hidden;
}
.video-wrapper iframe {
  position: absolute;
  top: 0; left: 0;
  width: 100%; height: 100%;
}

6. 流式排版（Fluid Typography）

使用 calc() 结合 rem 和 vw 单位，可以让字号随视口宽度平滑缩放，
无需在每个断点重复设置 font-size。这比在媒体查询中逐个调整更优雅。

这段标题会随窗口缩放

这段正文也会平滑变化。拖动浏览器窗口边缘，观察字号如何从移动端的小尺寸过渡到桌面端的大尺寸，无需任何断点切换。

/* ── 流式排版 ── */
h1 {
  /* 最小 1.5rem，随视口增加 4vw */
  font-size: calc(1.5rem + 4vw);
}

p {
  font-size: calc(0.875rem + 0.5vw);
}

/* ── 配合 clamp() 更简洁（CSS3）── */
h1 {
  font-size: clamp(1.5rem, 4vw + 0.5rem, 3.5rem);
  /*        最小值   理想值            最大值 */
}

/* ── Viewport 单位 ── */
vw  视口宽度的 1%  (1vw = 视口宽度 / 100)
vh  视口高度的 1%  (1vh = 视口高度 / 100)
vmin 取 vw 和 vh 中较小的值
vmax 取 vw 和 vh 中较大的值

7. Viewport 单位

vw、vh、vmin、vmax 是相对视口尺寸的单位，
常用于全屏布局、响应式间距和流式排版。注意不要单独用 vw 设置文字大小，会影响可缩放性。
3vw
3vh
1.2rem

对比：vw 随窗口变化 · vh 随窗口高度变化 · rem 固定不变 

/* Viewport 单位示例 */
.full-screen-section {
  width: 100vw;     /* 满视口宽度 */
  min-height: 100vh; /* 至少满视口高度 */
  padding: 5vw;     /* 与视口成比例的内边距 */
}

/* vmin 示例：取较小的视口维度 */
.icon {
  font-size: 10vmin; /* 在手机竖屏时跟随宽度，在横屏时跟随高度 */
}

/* 全屏背景 */
.hero {
  min-height: 100vh;
  display: flex;
  align-items: center;
  justify-content: center;
}

8. 响应式导航栏

桌面端水平排列导航链接，移动端折叠为汉堡菜单按钮。通过媒体查询切换布局。
点击下方示例中的 ☰ 按钮可切换导航展开/收起。
MySite
  • 首页
  • 产品
  • 关于我们
  • 博客
  • 联系方式
/* ── 桌面端：水平导航 ── */
.nav { display: flex; align-items: center; }
.nav-links { display: flex; gap: 8px; list-style: none; }
.hamburger { display: none; }  /* 隐藏菜单按钮 */

/* ── 移动端：汉堡菜单 ── */
@media (max-width: 600px) {
  .hamburger { display: block; }   /* 显示菜单按钮 */
  .nav-links {
    display: none;                 /* 默认隐藏链接 */
    flex-direction: column;        /* 纵向排列 */
    width: 100%;
  }
  .nav-links.active { display: flex; }  /* JS 切换 */
}

/* ── 纯 CSS 方案（使用 :target 或 checkbox hack）── */
.nav-toggle:checked ~ .nav-links { display: flex; }
.nav-toggle:checked ~ .hamburger .open { display: none; }
.nav-toggle:checked ~ .hamburger .close { display: block; }

9. 响应式表格

表格在小屏幕上容易溢出。常用方案：外层 overflow-x: auto 实现横向滚动，
或将表格转换为卡片列表。
技术 作用 适用场景 学习难度
Media Queries 根据屏幕宽度切换样式 所有响应式页面 ⭐
CSS Grid 二维网格布局 复杂页面骨架 ⭐⭐
Flexbox 一维弹性布局 导航栏、卡片行 ⭐
Viewport 单位 相对视口的尺寸单位 全屏、流式排版 ⭐
clamp() 限制值在最小与最大之间 流式排版 ⭐
<picture> 根据条件加载不同图片 响应式图片 ⭐⭐ 
/* 方案 1：横向滚动 */
.table-wrapper {
  width: 100%;
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
}
table { min-width: 600px; }

/* 方案 2：小屏变卡片 */
@media (max-width: 600px) {
  table, thead, tbody, th, td, tr { display: block; }
  thead { display: none; }
  td {
    padding: 8px 14px;
    position: relative;
    padding-left: 50%;
  }
  td::before {
    content: attr(data-label);
    position: absolute;
    left: 14px;
    font-weight: 700;
  }
}

10. 完整响应式布局示例

一个典型的响应式页面骨架：Header + Nav + Main + Sidebar + Footer。
移动端单列堆叠，桌面端三栏并排。拖动窗口观察布局切换。
Header — 网站头部
Nav — 导航栏
Main — 主内容区
这里放置文章、列表、表单等主要内容。在移动端占据全部宽度，在桌面端只占中间一列。
Aside — 侧边栏
相关链接、广告、标签云等辅助内容。
Footer — 页脚 © 2024
/* ── 移动优先的完整布局 ── */
.layout {
  display: grid;
  grid-template-areas:
    "header"
    "nav"
    "main"
    "aside"
    "footer";
  gap: 10px;
}

.layout .header  { grid-area: header; }
.layout .nav     { grid-area: nav; }
.layout .main    { grid-area: main; }
.layout .aside   { grid-area: aside; }
.layout .footer  { grid-area: footer; }

/* ── 桌面端切换为三栏 ── */
@media (min-width: 768px) {
  .layout {
    grid-template-columns: 200px 1fr 180px;
    grid-template-areas:
      "header header header"
      "nav    main   aside"
      "footer footer footer";
  }
}

12. 响应式设计最佳实践

总结响应式设计的关键原则和常见陷阱。

✅ 移动优先

先写手机样式，用 min-width 向上增强。更少的覆盖，更清晰的代码。

✅ 用相对单位

rem、em、vw、%、fr。避免固定 px，让布局自然流动。

✅ 流式图片

img { max-width: 100%; height: auto; } 一行代码解决 90% 的图片溢出问题。

❌ 隐藏内容

不要因为小屏就隐藏重要信息。重新排列内容，而不是删除它。

❌ 固定宽度容器

width: 960px 会让移动端溢出。用 max-width + 百分比替代。

❌ 只测一个设备

用 Chrome DevTools 的设备模拟测试多种尺寸，不要只看手机。

/* ── 推荐的响应式 CSS Reset 片段 ── */

  box-sizing: border-box;
}

img, picture, video, canvas, svg {
  max-width: 100%;
  height: auto;
}
