06 - 语义化结构元素

语义化标签让 HTML 结构更清晰，利于 SEO、可访问性和代码可读性。

1. 完整页面结构演示

一个典型的语义化页面包含 header、nav、main、article、section、aside、footer。

网站标题

<header> - 页面或区块的头部

首页 | 关于 | 联系
<nav> - 导航区域

<main> - 页面主要内容

文章标题

<article> - 独立完整的内容块（文章、帖子、卡片等）

章节 1

<section> - 文档中的逻辑分区

章节 2

每个 section 通常有自己的标题。

<aside> - 侧边栏内容（与主内容间接相关）

<address> - 联系信息：上海市浦东新区 | contact@example.com

<footer> - 页面底部（版权、链接等）

2. 语义化代码结构

<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>页面标题</title>
</head>
<body>
  <header>
    <h1>网站标题</h1>
    <nav>
      <a href="/">首页</a>
      <a href="/about">关于</a>
    </nav>
  </header>

  <main>
    <article>
      <h2>文章标题</h2>
      <section>
        <h3>章节标题</h3>
        <p>章节内容</p>
      </section>
    </article>

    <aside>
      <h3>侧边栏</h3>
      <p>相关链接或信息</p>
    </aside>
  </main>

  <footer>
    <p>&copy; 2024 版权所有</p>
  </footer>
</body>
</html>

3. 各语义化标签详解

标签 语义 典型用途
<header> 头部/介绍区 页面顶部、文章头部、区块头部
<nav> 导航 主导航菜单、面包屑、页内导航
<main> 主要内容 页面唯一的主要内容区域
<article> 独立内容 博客文章、新闻报道、用户评论
<section> 逻辑分区 章节、选项卡面板、功能区
<aside> 辅助内容 侧边栏、广告、引用框
<footer> 底部 版权信息、联系方式、底部链接
<address> 联系信息 作者信息、组织联系方式
<hgroup> 标题组 标题 + 副标题组合
<details> 可折叠内容 FAQ 手风琴、可展开区块
<summary> 摘要/标题 <details> 的可点击标题
<dialog> 对话框 模态弹窗、确认框

4. <details> 和 <summary> 可折叠内容

点击展开：什么是 HTML？

HTML（HyperText Markup Language）即超文本标记语言，是构建网页的标准语言。它使用标签来定义网页的结构和内容。

默认展开：什么是语义化？

语义化是指使用恰当的 HTML 标签来表达内容的含义。例如用 <nav> 表示导航，用 <article> 表示文章，而不是全部使用 <div>。

<details>
  <summary>点击展开标题</summary>
  <p>这里是隐藏的内容</p>
</details>

<details open>  <!-- open 属性默认展开 -->
  <summary>默认展开</summary>
  <p>可见的内容</p>
</details>

提示信息
