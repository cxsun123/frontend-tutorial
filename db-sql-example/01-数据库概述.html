01 - 数据库概述与基本概念

什么是数据库？

数据库（Database）是按照数据结构来组织、存储和管理数据的仓库。数据库管理系统（DBMS）是用于管理数据库的计算机软件系统。

为什么需要 DBMS？

相比文本文件或 Excel，DBMS 的优势

  • 多人共享 — 支持多用户并发访问和操作
  • 海量存储 — 处理 GB/TB 级别的数据
  • 数据一致性 — 通过事务和约束保证数据正确
  • 安全性 — 用户权限管理、数据加密
  • 故障恢复 — 备份和恢复机制
  • 可编程 — 存储过程、触发器、函数

核心术语

数据库领域的基本概念

  • 数据库（Database） — 保存有组织的数据的容器
  • 数据表（Table） — 某种特定类型数据的结构化清单
  • 模式（Schema） — 关于数据库和表的布局及特性的信息
  • 列（Column） — 表中的一个字段，定义数据类型
  • 行（Row） — 表中的一个记录（Record）
  • 主键（Primary Key） — 唯一标识表中每一行的列或列组合
  • 外键（Foreign Key） — 指向另一张表主键的列，建立表间关系
  • 索引（Index） — 加速数据检索的数据结构
  • 视图（View） — 基于 SQL 查询的虚拟表

DBMS 的发展历程

数据库管理系统的演进

  • 1960s — 层次数据库（IMS）和网状数据库
  • 1970s — 关系数据库诞生（IBM System R, Oracle）
  • 1980s — SQL 成为标准，RDBMS 商业化（DB2, SQL Server）
  • 1990s — 开源数据库兴起（MySQL, PostgreSQL）
  • 2000s — NoSQL 运动（MongoDB, Redis, Cassandra）
  • 2010s至今 — NewSQL、云数据库（Spanner, Aurora, TiDB）

DBMS 分类

根据数据模型分类

类型 代表产品 适用场景
关系型 (RDBMS) PostgreSQL, MySQL, Oracle 传统业务、金融、ERP
文档型 (Document) MongoDB, Couchbase CMS、博客、日志
键值型 (Key-Value) Redis, DynamoDB 缓存、会话、购物车
列族型 (Column-Family) Cassandra, HBase 大数据、时序数据
图数据库 (Graph) Neo4j, JanusGraph 社交网络、推荐引擎

SQL 语言分类

SQL（Structured Query Language）分为四大子语言

分类 全称 常用语句
DDL Data Definition Language CREATE, ALTER, DROP, TRUNCATE
DML Data Manipulation Language SELECT, INSERT, UPDATE, DELETE
DCL Data Control Language GRANT, REVOKE
TCL Transaction Control Language COMMIT, ROLLBACK, SAVEPOINT
实际使用中，90% 的 SQL 语句属于 DML（数据操纵语言），而 SELECT 是使用最频繁的操作。

SQL 语法要点

写 SQL 时需要注意的基本规则

  • SQL 语句不区分大小写，但习惯上关键字大写
  • 多条 SQL 语句以分号（;）分隔
  • 处理时所有空格都被忽略，可换行书写
  • 注释：-- 单行注释，/* */ 多行注释
-- 查询所有用户
SELECT id, name, email
FROM users
WHERE age > 18
ORDER BY name ASC;

小结

  • 数据库是结构化存储的容器，DBMS 是管理数据库的软件
  • 核心术语：表、行、列、主键、外键、索引、视图
  • DBMS 从层次→关系→NoSQL→NewSQL 不断发展
  • SQL 分为 DDL / DML / DCL / TCL 四大类
  • 不区分大小写，但推荐关键字大写增强可读性
