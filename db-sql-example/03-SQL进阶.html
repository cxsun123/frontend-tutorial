03 - SQL 进阶

SQL 进阶 — JOIN、聚合、子查询

掌握多表关联、数据聚合和子查询，是写好 SQL 的关键。

示例表

本章使用两张关联表

CREATE TABLE Customers (
    customer_id CHAR(4) PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    city        VARCHAR(50)
);

CREATE TABLE Orders (
    order_id    CHAR(4) PRIMARY KEY,
    customer_id CHAR(4) REFERENCES Customers(customer_id),
    amount      INTEGER NOT NULL,
    order_date  DATE
);

JOIN — 多表关联

将多张表的数据组合在一起

-- INNER JOIN：只返回匹配的行
SELECT c.name, o.order_id, o.amount
FROM Customers c
INNER JOIN Orders o ON c.customer_id = o.customer_id;

-- LEFT JOIN：左表所有行保留
SELECT c.name, o.order_id, o.amount
FROM Customers c
LEFT JOIN Orders o ON c.customer_id = o.customer_id;
-- 赵六没有订单，但也会显示，订单列为 NULL

-- RIGHT JOIN：右表所有行保留
SELECT c.name, o.order_id, o.amount
FROM Customers c
RIGHT JOIN Orders o ON c.customer_id = o.customer_id;

-- FULL OUTER JOIN：两表所有行保留
SELECT c.name, o.order_id, o.amount
FROM Customers c
FULL OUTER JOIN Orders o ON c.customer_id = o.customer_id;
MySQL 不支持 FULL OUTER JOIN，可用 UNION 实现。

聚合函数

对多行数据进行汇总计算

-- 常用聚合函数
SELECT COUNT(*)       AS 订单总数 FROM Orders;
SELECT SUM(amount)    AS 总金额  FROM Orders;
SELECT AVG(amount)    AS 平均金额 FROM Orders;
SELECT MAX(amount)    AS 最大金额 FROM Orders;
SELECT MIN(amount)    AS 最小金额 FROM Orders;

-- COUNT 的不同用法
SELECT COUNT(*)      FROM Orders;   -- 6 行
SELECT COUNT(amount) FROM Orders;   -- 6 行
SELECT COUNT(DISTINCT customer_id)
  FROM Orders;                      -- 3 个不同用户

GROUP BY — 分组

将数据按照某列分组后聚合

-- 按客户分组统计订单
SELECT customer_id,
       COUNT(*) AS 订单数,
       SUM(amount) AS 总金额,
       AVG(amount) AS 平均金额
FROM Orders
GROUP BY customer_id;

-- GROUP BY + WHERE
SELECT customer_id, SUM(amount) AS 总金额
FROM Orders
WHERE order_date >= '2024-03-01'
GROUP BY customer_id;

HAVING — 分组后过滤

WHERE 过滤行，HAVING 过滤组

-- 筛选总金额超过 2000 的客户
SELECT customer_id, SUM(amount) AS 总金额
FROM Orders
GROUP BY customer_id
HAVING SUM(amount) > 2000;

-- WHERE vs HAVING 的执行顺序
-- FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
WHERE 在分组前过滤行，HAVING 在分组后过滤组。HAVING 中可以使用聚合函数，WHERE 不行。

子查询

在查询中嵌套另一个查询

-- 标量子查询（返回单个值）
SELECT product_name, price
FROM Product
WHERE price > (SELECT AVG(price) FROM Product);

-- 列子查询（返回一列）
SELECT name FROM Customers
WHERE customer_id IN (
    SELECT customer_id FROM Orders
    WHERE amount > 1000
);

-- 关联子查询（引用外层列）
SELECT product_name, category, price
FROM Product p1
WHERE price > (
    SELECT AVG(price) FROM Product p2
    WHERE p2.category = p1.category
);

集合运算

对多个查询结果进行集合操作

-- UNION：并集（去重）
SELECT name FROM Customers
UNION
SELECT name FROM Employees;

-- UNION ALL：并集（不去重）
SELECT city FROM Customers
UNION ALL
SELECT city FROM Employees;

-- INTERSECT：交集
SELECT customer_id FROM Orders
INTERSECT
SELECT customer_id FROM Customers;

-- EXCEPT：差集
SELECT customer_id FROM Customers
EXCEPT
SELECT customer_id FROM Orders;

CASE 表达式

在 SQL 中实现条件分支

-- 简单 CASE
SELECT product_name,
    CASE category
        WHEN '衣物'   THEN '服装类'
        WHEN '厨房'   THEN '厨具类'
        WHEN '办公'   THEN '办公类'
        ELSE '其他'
    END AS 分类名称
FROM Product;

-- 搜索 CASE
SELECT product_name, price,
    CASE
        WHEN price < 1000  THEN '便宜'
        WHEN price < 3000  THEN '中等'
        WHEN price < 5000  THEN '偏贵'
        ELSE '高端'
    END AS 价格档次
FROM Product;

小结

  • JOIN 将多表数据关联，INNER/LEFT/RIGHT/FULL 四种
  • 聚合函数：COUNT / SUM / AVG / MAX / MIN
  • GROUP BY 分组聚合，HAVING 过滤组
  • 子查询可嵌套在 SELECT/FROM/WHERE 中
  • UNION/INTERSECT/EXCEPT 进行集合运算
  • CASE 表达式实现 SQL 内的条件分支
