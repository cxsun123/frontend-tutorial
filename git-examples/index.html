Git 入门 教程

什么是 Git？

Git 是一款强大的分布式版本控制系统。它能精准记录每一位团队成员的操作轨迹，让每一次编辑都更加安心、高效。

git init git add git commit git push

1. 安装 Git

不同操作系统的安装方式

# macOS（自带 Git，或使用 Homebrew）
brew install git

# Windows：从官网下载安装包
# https://git-scm.com/download/win

# Linux
sudo apt-get install git    # Debian/Ubuntu
sudo yum install git        # CentOS

# 验证安装
git --version

2. 基础配置

配置用户名和邮箱（必配）

# 配置用户信息（global 级别，对所有仓库生效）
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

# 查看配置
git config --list

3. 初始化仓库

将目录纳入 Git 版本管理

# 初始化本地仓库
git init

# 克隆远程仓库
git clone https://github.com/user/repo.git

4. .gitignore 文件

排除不需要版本控制的文件

# 常用通配符
*.log          # 忽略所有日志文件
*.pyc          # 忽略 Python 编译文件
node_modules/  # 忽略 node_modules 目录
dist/          # 忽略构建输出目录
.DS_Store      # 忽略 macOS 系统文件

# 示例 .gitignore
node_modules/
dist/
*.log
.env
.DS_Store

5. 工作区、暂存区、本地仓库

Git 的三个核心区域

# 工作区 -> 暂存区
git add file.txt
git add .              # 添加所有文件

# 暂存区 -> 本地仓库
git commit -m "提交信息"

# 查看状态
git status

# 查看历史
git log
git log --oneline     # 简洁模式

6. 分支操作

创建、切换、合并分支

# 创建分支
git branch feature

# 切换分支
git checkout feature
git switch feature      # 新语法

# 创建并切换分支
git checkout -b feature
git switch -c feature   # 新语法

# 查看分支
git branch              # 本地分支
git branch -a           # 所有分支

# 删除分支
git branch -d feature   # 删除已合并分支
git branch -D feature   # 强制删除

7. 远程仓库操作

与远程仓库同步代码

# 添加远程仓库
git remote add origin https://github.com/user/repo.git

# 推送到远程
git push origin main
git push -u origin main  # 首次推送并关联

# 拉取代码
git pull origin main

# 查看远程仓库
git remote -v

8. 分支合并

将一个分支的修改融入另一个分支

# 合并分支
git checkout main
git merge feature

# 快进合并（--ff）
git merge --ff feature

# 非快进合并（保留分支历史）
git merge --no-ff feature -m "合并 feature 分支"
提示：Fast-forward 合并不会创建新的提交，而非快进合并会保留分支拓扑结构。

9. 解决冲突

当两个分支修改同一文件时

# 冲突标记
<<<<<<< HEAD
当前分支的修改
=======
要合并分支的修改
>>>>>>> feature

# 解决步骤
# 1. 手动编辑文件，保留需要的代码
# 2. 删除冲突标记
# 3. 标记为已解决
git add file.txt

# 4. 完成合并
git commit
注意：解决冲突后记得删除 <<<<<<<、=======、>>>>>>> 标记。

10. Rebase 变基

将一个分支的修改"移植"到另一个分支顶部

# 在特性分支上 rebase main
git checkout feature
git rebase main

# 交互式 rebase（修改最近 3 个提交）
git rebase -i HEAD~3

# 常用命令
# pick    - 使用该提交
# reword  - 修改提交信息
# squash  - 合并到前一个提交
# fixup   - 合并并丢弃日志
# drop    - 移除该提交
重要原则：不要在公共分支（如 main）上执行 rebase！

11. 撤销操作

reset、revert、stash

# 软重置（保留修改）
git reset --soft HEAD~1

# 硬重置（丢弃修改，慎用！）
git reset --hard HEAD~1

# 还原（创建新提交来撤销）
git revert commit-hash

# 临时存储修改
git stash
git stash pop
git stash list
提示：reset 移动 HEAD，revert 创建新提交。公共分支推荐用 revert。

12. 标签与 Reflog

打标签和事故恢复

# 打标签
git tag v1.0.0
git tag -a v1.0.0 -m "版本 1.0.0"
git push origin v1.0.0

# 查看标签
git tag

# Reflog（查看所有操作历史）
git reflog

# 使用 reflog 恢复误删的提交
git reset --hard HEAD@{1}
提示：Reflog 是安全网，即使执行了 hard reset 也能找回丢失的提交（90天内）。

常用命令速查

日常开发必备命令

# 查看状态
git status

# 查看差异
git diff              # 工作区 vs 暂存区
git diff --cached     # 暂存区 vs 本地仓库

# 修改最后一次提交
git commit --amend

# 查看文件修改历史
git log file.txt
git blame file.txt
