07 - Next.js 入门

Next.js — React 的全栈框架

Next.js 是基于 React 的全栈框架，提供服务端渲染 (SSR)、静态生成 (SSG)、文件路由、API Routes 等开箱即用的功能。

为什么选择 Next.js？

相比纯 React 的优势

  • 更好的 SEO — SSR/SSG 输出完整的 HTML，搜索引擎可抓取
  • 更快的首屏加载 — 服务端生成 HTML，减少客户端 JS 执行
  • 文件系统路由 — 基于 pages/ 或 app/ 目录自动生成路由
  • 内置 API 层 — 在 Next.js 中直接编写后端 API
  • 图片优化 — next/image 自动优化、懒加载、WebP 格式
  • 全栈能力 — 一个项目搞定前后端

SSR vs SSG vs CSR

三种渲染策略的对比

// 1. SSR — 每次请求在服务端渲染
// 页面 HTML 在每次请求时动态生成，SEO 好，数据实时
export async function getServerSideProps() {
  const res = await fetch('https://api.example.com/posts');
  const posts = await res.json();
  return { props: { posts } };
}

// 2. SSG — 构建时生成静态 HTML
// 页面在 build 时生成，速度快，适合不常变的内容
export async function getStaticProps() {
  const res = await fetch('https://api.example.com/posts');
  const posts = await res.json();
  return { props: { posts } };
}

// 3. CSR — 客户端渲染（纯 React 方式）
// 初始 HTML 为空壳，JS 加载后渲染内容
function Page() {
  const [data, setData] = useState(null);
  useEffect(() => {
    fetch('/api/data').then(r => r.json()).then(setData);
  }, []);
  return <div>{data?.content}</div>;
}

创建 Next.js 项目

使用 create-next-app 快速初始化

# 创建项目（推荐使用 App Router）
npx create-next-app@latest my-next-app

# 进入目录并启动
cd my-next-app
npm run dev
# 访问 http://localhost:3000

项目结构 (App Router)

Next.js 13+ 引入的 App Router 结构

my-next-app/
├── app/
│   ├── layout.jsx      # 根布局（所有页面共享）
│   ├── page.jsx        # 首页 /
│   ├── about/
│   │   └── page.jsx    # /about 页面
│   ├── blog/
│   │   ├── page.jsx    # /blog 列表页
│   │   └── [slug]/
│   │       └── page.jsx # /blog/hello-world 动态路由
│   └── api/
│       └── hello/
│           └── route.js # /api/hello 接口
├── public/             # 静态资源
├── package.json
└── next.config.js

根布局

app/layout.jsx 定义所有页面的公共结构

// app/layout.jsx
export const metadata = {
  title: '我的 Next.js 应用',
  description: '使用 Next.js 构建的全栈应用',
};

export default function RootLayout({ children }) {
  return (
    <html lang="zh-CN">
      <body>
        <header>
          <nav>
            <a href="/">首页</a>
            <a href="/about">关于</a>
            <a href="/blog">博客</a>
          </nav>
        </header>
        <main>{children}</main>
        <footer>© 2024 我的应用</footer>
      </body>
    </html>
  );
}
Layout 在页面切换时保持状态，不会重新渲染。适合放导航栏、页脚、侧边栏等。

小结

  • Next.js 是 React 的全栈框架，解决 SEO 和首屏性能问题
  • SSR、SSG、CSR 三种渲染策略适用于不同场景
  • App Router 基于文件系统自动生成路由
  • Layout 提供了跨页面的公共模板
  • pages/ 目录中的每个文件对应一个路由
