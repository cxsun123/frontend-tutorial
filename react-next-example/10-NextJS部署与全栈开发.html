10 - Next.js 部署与全栈开发

从开发到上线

本章涵盖 Next.js 的部署、环境变量、中间件、认证集成等生产环境必备知识。

构建与部署

Next.js 项目构建后部署到各种平台

# 构建项目
npm run build

# 启动生产服务器
npm start

# 导出静态站点（纯静态部署）
npx next build && npx next export

推荐部署平台:

  • Vercel — Next.js 官方平台，一键部署，自动适配 SSR/SSG
  • Netlify — 支持静态导出，通过 @netlify/plugin-nextjs
  • Docker — 自定义容器部署到任意云平台

环境变量

管理不同环境下的配置

# .env.local — 本地开发（不上传到 git）
DATABASE_URL="postgresql://localhost:5432/mydb"
API_SECRET="my-secret-key"

# .env — 所有环境共享
NEXT_PUBLIC_SITE_URL="https://mysite.com"

# .env.production — 生产环境
# .env.development — 开发环境
// 在服务端组件中使用
export default async function Page() {
  // 服务端环境变量（直接访问）
  const dbUrl = process.env.DATABASE_URL;
  return <div>DB: {dbUrl}</div>;
}

// 'use client' 组件只能使用 NEXT_PUBLIC_ 前缀的变量
console.log(process.env.NEXT_PUBLIC_SITE_URL);
NEXT_PUBLIC_ 前缀的变量会在客户端暴露，不要放敏感信息。

Middleware 中间件

在请求到达页面之前执行逻辑

// middleware.js（在项目根目录）
import { NextResponse } from 'next/server';

export function middleware(request) {
  const token = request.cookies.get('token');

  // 未登录用户重定向到登录页
  if (!token && request.nextUrl.pathname.startsWith('/dashboard')) {
    return NextResponse.redirect(new URL('/login', request.url));
  }

  // 添加请求头
  const response = NextResponse.next();
  response.headers.set('x-custom-header', 'hello');
  return response;
}

// 匹配路径配置
export const config = {
  matcher: ['/dashboard/:path*', '/admin/:path*'],
};

认证集成

使用 NextAuth.js 添加认证

// app/api/auth/[...nextauth]/route.js
import NextAuth from 'next-auth';
import GithubProvider from 'next-auth/providers/github';

const handler = NextAuth({
  providers: [
    GithubProvider({
      clientId: process.env.GITHUB_ID,
      clientSecret: process.env.GITHUB_SECRET,
    }),
  ],
  callbacks: {
    async session({ session, token }) {
      session.user.id = token.sub;
      return session;
    },
  },
});

export { handler as GET, handler as POST };

// 在客户端组件中使用
'use client';
import { useSession, signIn, signOut } from 'next-auth/react';

export default function AuthButton() {
  const { data: session } = useSession();

  if (session) {
    return (
      <div>
        <p>欢迎, {session.user.name}</p>
        <button onClick={() => signOut()}>退出</button>
      </div>
    );
  }
  return <button onClick={() => signIn()}>登录</button>;
}

图片优化

next/image 提供自动优化

import Image from 'next/image';

export default function Page() {
  return (
    <div>
      {/* 本地图片 — 自动获取宽高 */}
      <Image
        src="/profile.jpg"
        alt="头像"
        width={200}
        height={200}
        priority // 首屏图片优先加载
      />

      {/* 远程图片 — 需配置域名 */}
      <Image
        src="https://example.com/photo.jpg"
        alt="远程图片"
        width={800}
        height={600}
        placeholder="blur" // 模糊占位
      />
    </div>
  );
}
// next.config.js
module.exports = {
  images: {
    domains: ['example.com'],
    formats: ['image/avif', 'image/webp'],
  },
};

// 静态 metadata
export const metadata = {

// 动态 metadata
export async function generateMetadata({ params }) {
  const post = await getPost(params.slug);
  return {
    title: post.title,
    description: post.excerpt,
    openGraph: {
      title: post.title,
      images: [post.coverImage],

小结

  • Vercel 是部署 Next.js 的最佳选择，一键部署
  • 环境变量通过 .env 文件管理，NEXT_PUBLIC_ 前缀暴露给客户端
  • Middleware 在路由层面执行认证、重定向、日志等逻辑
  • NextAuth.js 快速集成 OAuth 认证
  • next/image 自动优化图片，metadata API 提升 SEO
  • 恭喜！你已经掌握了 React + Next.js 全栈开发的核心知识！
